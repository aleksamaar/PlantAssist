### ⚠️ Правило №1: облако — единственный источник правды

И телефон, и компьютер работают **только с облаком**. Локальные `data.json`,
`sowings.json`, `photos/` — это **устаревший снимок до переезда**, они не
синхронизируются с облаком и не нужны.

- `start.bat` → открывает облако (то, что нужно всегда)
- `start_local.bat` → локальный сервер, **только для разработки**, данные там старые

## Что сделано

**Приложение** — учёт домашних растений: полив, подкормка, пересадка, обработка
от вредителей, витамины, цветение, черенки, посевы, архив, достижения (71 шт.),
BI-дашборд со статистикой.

**Облако** — развёрнуто на PythonAnywhere (бесплатный тариф), вход по паролю,
настоящий HTTPS. Работает без компьютера.

**Телефон** — PWA установлена как приложение (WebAPK), иконка 🌿 в списке
приложений, полноэкранный режим. Мобильный интерфейс переработан: нижняя
навигация 4 вкладки + «Ещё», модалки-шторки, тап-зоны 44px.

**Бэкапы** — `/api/backup` отдаёт ZIP (данные + фото), `/api/restore` принимает
обратно. Скрипт `backup.py` качает из облака в папку Яндекс Диска.

## 🐛 Известная проблема: Яндекс Диск ломает git

**Симптом:** после переноса папки Диска синхронизация удалила часть git-объектов
и файл `.git/index`. Проверено экспериментом: созданный `index` исчезал через 5 секунд.

**Лечение** (данные не теряются — код на GitHub, растения в облаке):

```bash
git fetch origin   # если пропали объекты ("bad object", "invalid sha1 pointer")
git reset          # если статус показывает все файлы как удалённые (D)
git fsck           # проверить, что всё чисто
```

**Риск:** может повториться при массовой переиндексации Диска. Раньше (годами, на
старом месте внутри Диска) проблем не было — вероятно, дело в разовом переезде.

Если начнёт повторяться — вынести проект из папки Диска, а бэкапы направить внутрь
Диска через переменную `PLANTASSIST_BACKUP_DIR` (уже поддерживается в `backup.py`).

## Как запустить и проверить

```bash
# Работа с приложением (облако)
start.bat

# Локальный сервер для разработки (данные старые!)
start_local.bat        # спросит подтверждение
py app.py              # то же самое напрямую

# Бэкап из облака в Яндекс Диск
backup.bat
```

Локально пароль не спрашивается — он включается только если задана переменная
`PLANTASSIST_PASSWORD` (она есть только на хостинге).

**Тестов и линтера нет** — всё проверяется руками в браузере.
После правки `app.py` сервер нужно перезапустить вручную.

## Обновить облако после изменений кода

```bash
# 1. Локально
git add -A && git commit -m "..." && git push

# 2. На PythonAnywhere → Consoles → Bash
cd PlantAssist && git pull

# 3. PythonAnywhere → вкладка Web → зелёная кнопка Reload
```

На телефоне может потребоваться смахнуть приложение из недавних — service worker
подтянет новые стили/скрипты (он отдаёт свежий код при наличии сети).

## Секреты и доступы

| Что | Где лежит | В git? |
|-----|-----------|--------|
| Пароль от приложения | `backup_config.bat` + WSGI-конфиг на PythonAnywhere | ❌ нет (в `.gitignore`) |
| Секрет сессий | WSGI-конфиг на PythonAnywhere (`PLANTASSIST_SECRET`) | ❌ нет |
| Ключ сессий (локально) | `.secret_key` (создаётся сам) | ❌ нет |
| Логин PythonAnywhere | `AleksaMaar` | — |

⚠️ **Репозиторий публичный.** Пароли писать только в `backup_config.bat`
(он в `.gitignore`), никогда в `backup.bat`, `start.bat` или код.

Конфиг на хостинге: **Web** → **WSGI configuration file**:
```python
import sys, os
path = '/home/AleksaMaar/PlantAssist'
if path not in sys.path:
    sys.path.insert(0, path)
os.environ['PLANTASSIST_PASSWORD'] = '...'
os.environ['PLANTASSIST_SECRET'] = '...'
from app import app as application
```
