# 📱 PlantAssist на телефоне (Android, PWA)

Приложение можно установить на телефон как обычное приложение и пользоваться по домашнему Wi-Fi. Компьютер при этом должен быть включён и запущен `app.py`.

## Один раз: настройка

### 1. Установить сертификат на телефон
Chrome требует доверенный HTTPS. У нас свой локальный сертификат.

1. Файл **`certs/rootCA.crt`** уже лежит в папке проекта внутри Яндекс Диска — он
   автоматически синхронизируется. Открой **приложение Яндекс Диск** на телефоне и
   скачай оттуда `rootCA.crt` (или перешли себе файл любым способом).
2. Открой на телефоне: **Настройки → Безопасность → Ещё → Шифрование и учётные данные
   → Установить сертификат → Сертификат ЦС** (названия могут отличаться по модели).
3. Выбери скачанный `rootCA.crt`. Понадобится PIN/пароль экрана.
4. Может появиться предупреждение «сеть может отслеживаться» — это нормально для
   своего сертификата, соглашайся.

### 2. Проверить, что телефон в той же Wi-Fi, что и компьютер.

## Каждый раз: запуск

1. На компьютере запусти `start.bat` (или `python app.py`). Сервер стартует по HTTPS.
2. На телефоне в **Chrome** открой: **https://192.168.1.14:5000**
   Должен быть замочек 🔒 (безопасно).
3. Меню Chrome (⋮) → **«Установить приложение»** / **«Добавить на главный экран»**.
4. Запускай с домашнего экрана — иконка 🌿, полноэкранный режим, как настоящее приложение.
   После первой загрузки работает даже при слабом интернете (кэш).

## Если что-то не так

- **Нет замочка / «небезопасно»** — сертификат не установлен или установлен не как «ЦС».
  Повтори шаг 1. Используй именно **Chrome** (Firefox не доверяет пользовательским CA).
- **Не открывается по IP** — проверь, что телефон и компьютер в одной Wi-Fi и сервер запущен.
- **IP компьютера изменился** (было `192.168.1.14`) — узнай новый IP и пересоздай серверный
  сертификат:
  ```bash
  bash gen-certs.sh 192.168.X.X
  ```
  Перезапусти сервер. Сертификат на телефоне переустанавливать **не нужно** — CA тот же.
  Совет: закрепи за компьютером статический IP в настройках роутера, чтобы не менялся.
- **Сертификат раз в год** — серверный сертификат действует 397 дней, потом просто снова
  запусти `gen-certs.sh`.

## Вернуться к обычному HTTP
Удали или переименуй папку `certs/` — сервер снова стартует по `http://localhost:5000`.
